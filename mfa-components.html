MFA Components / Docu

Overview

This is running a (gently) forked Zurb Foundation framework along with our SSO interface. The build process is (again) a fork of a related project Template, see the readme there. NPM, Bower, Gulp, Git Submodules - pretty modular / manageable.

Typography

It should "just work". Don't worry about pixel matching the visual comps, the framework wins (code > design artifacts). That said if something looks really off let me know and I can add a class to the framework.

Semantic headers - for content <h1> is at the top of the page describing what that page does. <h2> labels sections within a page (activated, unactivated, advanced). <h3> is used for device names/types within the <div class="device-info"> as featured on the MFA device page. For the sidebars/help <h1> appears in an offset yellow box, the rest follows in standard fashion.

Small disclaimer text at the bottom of pages is wrapped with <p class="note-that">standard rates apply</p>.

Layout

<section class="mfa"> is the main wrapper for the page (floats above the blue Los Angeles sky). <article class="content"> holds the main content, with <aside class="supporting-info"> holding the sidebar of help text on desktop/tablet view and being a toggle on mobile. It's a simple structure. :) The major structural elements require a div inside them because I can't swap singularity.gs/breakpoint into foundation due to name-space conflicts with media queries (without causing more trouble than it's worth).

Forms

Client side validation is done with Zurb's Abide. While their documentation is far more thorough, following the basic example below (error span, attributes on form element) is the basic template to follow. It extends HTML5 form validation in ways I find useful.

  <form data-abide novalidate>
    <label>
      <input type="checkbox" required>I have installed Duo Mobile on my device.
      <span class="form-error">You must have installed Duo Mobile to Continue.</span>
    </label>
    <button type="submit" class="primary-button">Sync Duo Mobile App</button> 
    <button type="submit" class="cancel-button">Cancel</button>
  </form>

Buttons

For inline buttons on device management, wrap in the classed fieldgroup. The button element on it's own gives the default button, .primary-button gives a solid blue primary choice, and .cancel-button the warningish cancel/back to break application flow.

  <fieldgroup class="inline">
    <button class="primary-button" type="submit">I'm a primary button look at me!</button>
    <button type="submit">I'm just a button.</button>
    <button class="cancel-button" type="submit">Think twice before clicking me.</button>
  </fieldgroup>

Banners

Alert is shown as an example on sample page with notification banner.

Usage is a container div, either <div class="banner-alerts"> for alerts/fails, or <div class="banner-success"> to notify of a successful operation. Message contents are with an un-classed <p> tag inside, the x to close in an un-classed <aside>.

About MFA in UCLA Logon

Multi-Factor Authentication (MFA) adds extra protection for your UCLA Logon account. With MFA enabled, you'll be asked to take an extra step to confirm your identity when signing in. This extra step prevents a hacker from accessing your account using a stolen password.
